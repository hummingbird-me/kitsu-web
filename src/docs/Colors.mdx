import { ColorItem, ColorPalette, Meta } from '@storybook/blocks';

import colors from 'app/styles/globals/colors.json';

export function generateColors(color) {
  return Object.fromEntries(
    Object.keys(colors['custom-properties'])
      .filter((key) => key.includes(color))
      .map((key) => [key, colors['custom-properties'][key]])
  );
}

# Colors

Kitsu has two "layers" of color palettes, starting with a set of base colors, which are then aliased
in each theme to a set of _semantic_ colors which are then used by components.

## Base Colors

The base color palette should generally _not_ be directly used by components, though exceptions can
be made. Usually the exceptions should be limited to declaring a color _group_ rather than the exact
color (e.g. `red` instead of `--red-2`) so that the component styles can account for the theme and
adjust the brightness accordingly to maintain proper contrast.

## Semantic Colors

Semantic colors are defined in each theme and are used by components. They are aliases to the base
colors, named to describe their purpose. For example, in the light theme `--page-background` is
`--grey-1` but in the dark theme it is `--kitsu-purple-5`.

To see more, please see the CSS files in `src/styles/themes/`.
